# Link Garden 项目记录

> 最后更新：2026-03-23 UTC

## 项目定位

Link Garden / 净界 是一个前后端分离的个人内容站项目，方向不是传统博客模板，而是：

- 个人博客门面
- 技术收藏站
- 外部文章卡片归档
- 本地 Markdown 文章创作与展示

当前主风格：

- 深色
- 冷调蓝青 / 蓝紫
- 克制
- 带一点赛博感
- 不做过重荧光、不过度装饰

## 已明确的设计结论

### 首页
- 首页只放三个大类卡片，不继续加复杂一级导航模块
- 英文 `LINK GARDEN` 已去掉
- Hero 文案当前为：
  - `是个人博客，也是技术收藏展厅`
- 不再做首页“单卡片自动放大/居中”等特殊规则
- 不做首页多卡片 2/3/4 篇特殊排布策略

### 文章详情页
- 顶部头图尽量接近个人博客样式：
  - 左下标题 + 一行信息
  - 信息密度克制
- 去掉返回按钮
- 去掉右侧目录浮卡
- 如果 Markdown 首行是 `# 一级标题`，详情页渲染时会剥掉，避免与头图标题重复

### 后台新增文章页
- 优先是创作工作台，不是表单说明页
- 标题在最上方
- 主体使用编辑器本身预览能力，不再重复手搓右侧预览
- 摘要、标签、封面等次要信息下沉到底部“附加信息”区

### 后台文章管理页
- 删除按钮暂时隐藏，避免误触
- “查看”已改成“下架”占位
- “回到前台”入口独立放在页面左上角

## 需要避免的坑

1. 不要再把后端跑在 debug/reloader 模式下做持久服务
2. 改后端 Python 文件后，先检查语法，不然 systemd 会起不来
3. 改详情页 CSS 时注意不要让 `.markdown-body h1/h2/h3` 把 `.article-prose` 的单独颜色覆盖掉
4. 详情页代码卡片包装逻辑不要重复执行，否则容易把 `pre` 整块删掉
5. 新文章海报尽量统一转成本地 `/covers/<id>.png`，不要长期依赖外链

---
